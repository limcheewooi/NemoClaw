## Background

v0.0.1 (shipped 2026-04-21) used the vanilla `v<X.Y.Z>` tag scheme. When pushing the canonical `v0.0.1` tag at Step 5, the release collided with upstream NVIDIA/NemoClaw, which already holds `v0.0.1` through at least `v0.0.20` on its own commits. We resolved v0.0.1 by deleting the fetched upstream tag locally and pushing our own to origin, but the broader collision remained: every future fork release (v0.0.2, v0.0.3, …) would hit the same problem, and every `git fetch upstream --tags` would produce collision noise.

## Reason

User decision on 2026-04-21: adopt option (B) from the three options raised — namespace all fork release tags with an `AGTF-v` prefix. This fully separates the fork's release tags from upstream's tag space, eliminating collisions on `git fetch upstream --tags` and making it unambiguous which release belongs to which project when reading `git tag -l` output. v0.0.1 keeps its historical unnamespaced tags; v0.0.2 onward uses the new prefix.
